## 2. 잘 되어 있는 것 (건드리지 말 것)

### ✅ 캐릭터 패시브 시스템
`characterWeapon` 기반 분기가 BattleScene 전반에 걸쳐 잘 구현되어 있다.  
Guardian의 매 턴 쉴드 +5, Lancer의 투창 즉사 조건, Berserker의 HP 역전 시 폭발력 — 이것들은 실제로 플레이 스타일을 바꾼다. **이 시스템의 방향은 맞다.**

### ✅ 카드 고유 효과
속성별 테마(취약·화상·연쇄·방어막+방깎·회복+방어막)가 명확하고 실제 전략을 만들어낸다.  
"불 카드로 화상 걸고 → 번개 연쇄로 폭딜" 같은 콤보가 진짜 작동한다.

### ✅ 최종 보스 3페이즈
5턴 제한 + 페이즈별 패턴 변화 + 쉴드 리셋 구조는 잘 설계된 클라이맥스다.  
페이즈 3의 "매 턴 ATK +4 + 짝수 턴 이중 타격"은 점점 올라가는 압박감이 좋다.

### ✅ 적 행동 예고 UI
`enemyIntentCont`로 다음 턴 피해를 미리 보여주는 것은 "이번 턴 방어할지 말지"의 판단 근거가 된다.

### ✅ 노드 이벤트 다양성
10종류의 이벤트 중 HEART(HP 희생 vs 카드밸류), INDIAN_POKER(덱 정제), CARD_FLIP(도박) 등은 트레이드오프가 있는 좋은 설계다.

## 3. 핵심 문제 (우선순위 순)

---

### 🔴 문제 1 — 적이 아무 생각 없이 때린다

현재 일반 적의 전투 로직:
```
플레이어 공격 → 적이 고정 ATK로 반격 → 반복
```

속성이 다른 10종의 적이 있지만 **전투에서 체감되는 차이가 없다.**  
물 속성 적이든 불 속성 적이든 행동 패턴이 동일하다: "매 턴 ATK만큼 때린다."

적이 어떤 속성인지는 오직 상성 계산에만 쓰인다. 적이 자기 속성을 "사용"하지 않는다.

**결과:** 일반 전투는 "이번 턴 무슨 카드 쓸까"가 전부다. "이번에 적이 어떻게 행동할 것인가"를 고려할 필요가 없다.

---

### 🔴 문제 2 — 골드가 쌓이기만 하고 쓸 데가 없다

골드를 버는 방법: 전투 승리, CARD_FLIP 도박, 이벤트 보상  
골드를 쓰는 방법: **없다**

CARD_FLIP 이벤트에서 도박에 베팅할 수 있지만, 이건 골드를 "쓰는" 게 아니라 "걸고 더 받거나 잃는" 것이다.  
`currentGold`는 전달되고 저장되고 표시도 되지만, 실질적인 소비처가 없다.

**결과:** 골드를 많이 모아도 아무 의미가 없다. "이번에 골드를 아껴서 상인한테 좋은 카드 사야지"라는 목표가 생기지 않는다.

---

### 🔴 문제 3 — 방어 카드를 쓸 이유가 부족하다

방어(defense) 카드로 생성된 쉴드는 **해당 턴에만 유효하다.**  
적이 이번 턴 10 데미지를 줄 예정인데 쉴드를 20 쌓으면 10은 버려진다.

더 큰 문제: **공격이 항상 이득이다.**

카드 3장 사용 후 플레이어는 `executePlayerBaseAttack()`으로 ATK만큼 자동 공격을 한다.  
방어 카드를 쓰면 이 자동 공격은 여전히 나가지만, 공격 카드로 교체하면 추가 데미지도 넣을 수 있다.  
방어막이 생존을 보장하는 상황(HP가 위험할 때)이 아니면 공격이 거의 항상 우월하다.

**결과:** 덱에서 방어 카드의 가치가 낮다. Guardian의 방어 카드 2배 패시브가 있어도 마찬가지다.

---

### 🟡 문제 4 — 적 강도가 고정되어 있다

`completedElements` (클리어한 속성 맵 목록)가 저장은 되지만, **실제로 적 스탯 스케일링에 전혀 사용되지 않는다.**

CLAUDE.md의 미구현 TODO에도 명시된 내용이다:
> "적 맵 진행 스케일링: completedElements.length를 기반으로 맵 이동 시마다 적 스탯 강화"

게임 초반에 먹은 ENHANCE, TREASURE, STAR_UP 이벤트 덕분에 플레이어는 점점 강해지는데,  
적은 항상 같은 강도다. **후반 갈수록 적이 더 쉬워지는 역설적 난이도 곡선이 된다.**

---

### 🟡 문제 5 — 덱이 너무 두껍고 얇아질 방법이 없다

초기 덱 크기: 25~35장 (무기 타입에 따라 다름)  
턴당 드로우: 5장  
20턴 내에 보는 카드 수: 최대 100장 (=덱을 약 3~4회 순환)

카드 제거 수단이 없다. INDIAN_POKER(덱 정제)가 있지만 이것도 카드를 교환하는 개념이지 제거가 아니다.

**결과:** 원하는 카드를 원할 때 뽑는 "덱 구축"의 재미가 없다. 덱은 항상 두꺼운 채로 운에 의존한다.

---

### 🟡 문제 6 — 런 사이에 달라지는 게 없다

런을 완주하거나 실패해도 다음 런이 달라지지 않는다.  
`legacy.json`에 `totalGold`, `allEquipment`, `runs`가 저장되지만, 게임플레이에 영향을 주는 메타 진행이 없다.

CharacterSelectScene에서 이전 런 장비를 시작 장비로 선택하는 기능이 UI에 있지만, CLAUDE.md에는 "Legacy 장비 선택 미구현"으로 표시되어 있다.

---

## 4. 밸런스 진단

### 캐릭터 밸런스

| 캐릭터 | 강점 | 약점 | 현재 평가 |
|--------|------|------|-----------|
| Guardian | 쉴드 2배, 매 턴 +5, 높은 HP·DEF | 낮은 ATK, 방어 의존형 | 🟡 약함 — 방어가 공격보다 약해서 손해 |
| Ranger | 화살+ATK 공식, 연쇄 +50% | 낮은 HP, DEF 0 | ✅ 강함 — 번개 덱과 시너지 극대화 |
| Berserker | HP <60% 폭발력 | 중간 HP, 초반 약함 | 🟡 불안정 — 초반 전투에서 패시브 발동 어려움 |
| Titan | 항상 관통 | DEF 0, 높은 HP라 보상 | ✅ 강함 — 관통이 후반 보스에 매우 유리 |
| Lancer | 투창 항상 크리, 즉사 조건 | DEF 0, 낮은 ATK | 🟡 투창 카드 뽑기 운 의존도 높음 |

**Guardian이 체감상 가장 약하다.** 방어막이 1턴짜리라는 구조적 문제가 Guardian 컨셉과 정면으로 충돌한다.

---

### 카드 밸런스

속성 카드 간 유불리:

- **번개(lightning)** — Ranger와 함께 쓰면 chain 효과 + Ranger 패시브 50% 보너스로 연쇄 데미지가 매우 강하다
- **불(fire)** — 화상은 보스전에서 BOSS_FINAL에 적용되지 않아서 가장 강한 적에게 무효
- **풀(grass)** — 공격력이 낮고 회복 효과는 불확실한 상황에서만 유효
- **물(water) 4성** — 취약 2턴 + 다중타격(2회). 현재 가장 강력한 단일 카드 중 하나
- **땅(earth) 5성** — 방어막 25 + 방깎 10. Guardian과 조합 시 매우 강하지만 Guardian 자체가 약해서 아이러니

---

### 이벤트 밸런스

| 이벤트 | 체감 가치 | 평가 |
|--------|-----------|------|
| HEART (생명력 제단) | ★★★★★ | 최강. HP -5로 카드밸류 +5%는 항상 이득 |
| STAR_UP (별 강화) | ★★★★☆ | 강함. 고성급 카드 = 강한 효과 확보 |
| TREASURE (보물) | ★★★★☆ | 장비 특수효과에 따라 게임 판도 바뀜 |
| ENHANCE (강화) | ★★★☆☆ | 안정적이나 劇的 변화 없음 |
| INDIAN_POKER (덱 정제) | ★★★☆☆ | 덱 교환이지 제거가 아니라서 임팩트 제한 |
| CARD_SWAP (카드 교환) | ★★☆☆☆ | 원하는 방향으로 바꾸기 어려움 |
| CARD_FLIP (도박) | ★★☆☆☆ | 완전 운. 기대값이 낮게 느껴짐 |
| SHIELD_UP (회복의 샘) | ★★☆☆☆ | shieldMult +0.2 vs HP 28% 회복. 대부분 HP 회복이 나음 |

**HEART 이벤트가 지나치게 강하다.** -5 HP로 +5% 카드밸류를 받는 것은 거의 항상 최선의 선택이다.  
게임 후반부에 HEART를 여러 번 밟으면 카드 배율이 1.3~1.5배까지 쌓여서 적이 순식간에 녹는다.

## 5. 개선 우선순위

### 🔴 즉시 해야 할 것 (재미를 직접 높임)

#### 1. 골드 소비처 추가 — 상인 노드
```
NODE_TYPE.MERCHANT 추가
- 카드 제거: 50 Gold (노말/수비 카드 제거 → 덱 압축)
- 카드 구매: 80-150 Gold (3~4성 카드 3장 중 선택)
```
골드가 의미 있어지면 "골드를 아껴야 하나 써야 하나"의 결정이 생긴다.

#### 2. 적 속성별 고유 행동 추가
```typescript
// 최소한 이것만으로도 전투가 달라짐
water 적: 2턴마다 방어력 +5 (쌓이면 뚫기 어려워짐)
fire 적:  3턴에 한 번 화상 공격 (플레이어에게 화상 부여)
grass 적: HP 50% 이하 시 자기 회복
lightning 적: 기절 카운터 면역 (스턴이 통하지 않음)
earth 적: 매 턴 방어막 10 (쌓인 쉴드를 먼저 깎아야 함)
```
코드 변경은 `doEnemyTurn()`에 적 속성별 분기 추가뿐이다.

#### 3. HEART 이벤트 재조정
현재 "소 희생: -5 HP, +5%" 는 초반에도 항상 이득이다.  
HP 10 미만에서는 소 희생 비활성화하거나, 소 희생 비용을 -8 HP로 올려야 선택이 의미 있어진다.

---

### 🟡 단기 (게임을 훨씬 풍부하게)

#### 4. 적 스케일링 구현
`completedElements.length` 이미 추적 중. 이걸 쓰기만 하면 된다.
```typescript
// MainScene에서 BattleSceneData 생성 시
const scaleFactor = 1 + this.completedElements.length * 0.15;
enemyHp   *= scaleFactor;
enemyAtk  *= scaleFactor;
```

#### 5. 방어막 캐리 구조 수정
Guardian이 의미 있으려면 방어막이 1턴 이상 지속될 수 있어야 한다.  
"이번 턴 공격하지 않고 방어막을 쌓으면 2턴 연속 버틸 수 있다"는 전략이 가능해야 한다.
```typescript
// 옵션 A: 쉴드를 다음 턴까지 50% 유지
// 옵션 B: Guardian 전용 — 사용하지 않은 쉴드가 다음 턴으로 이월
```

#### 6. 덱 카드 제거 기능
INDIAN_POKER를 진짜 덱 압축 수단으로 만들거나, 상인 노드에서 카드 제거 제공.  
덱이 얇아지면 원하는 카드가 더 자주 나온다는 "빌드"의 재미가 생긴다.

---

### 🟢 중기 (장기 플레이 이유 만들기)

#### 7. 메타 진행 — legacy 활용
현재 `legacy.json`에 `totalGold`, `allEquipment`, `runs`가 쌓이는데 아무 쓸모가 없다.  
`runs` 횟수를 기반으로 새 이벤트 해금, 새 카드 해금 등 최소한의 메타 진행을 추가하면  
런 실패도 "기여"가 된다.

#### 8. CharacterSelectScene 시작 장비 선택
CLAUDE.md에도 미구현으로 명시된 항목.  
`allEquipment`에서 1개를 골라 시작하는 기능 — 메타 진행의 가장 직관적인 형태.

## 6. 코드 구조 제안

BattleScene의 `executeCardAction()`을 분리하면 이후 작업이 훨씬 쉬워진다.

```
BattleScene.ts (현재 2188줄)
  └── CardEffectProcessor.ts  (카드 효과 계산 전담)
  └── DamageCalculator.ts     (크리·상성·관통·연쇄 계산)
  └── EnemyAI.ts              (적 행동 패턴)
```

지금 당장 안 해도 되지만, **다음 번 카드 효과나 패시브 추가 시** 같이 리팩터링하면 좋다.

## 7. 총평 및 방향성

### 지금 이 게임은:
- 완성도 있게 돌아가는 로그라이크 덱빌딩 게임의 **프로토타입**
- 카드 전략, 캐릭터 패시브, 다양한 이벤트 — **뼈대가 잘 만들어져 있다**
- 최종 보스를 처음 만났을 때의 긴장감은 **진짜다**

### 아직 부족한 것:
- **"또 하고 싶다"는 이유** — 각 런이 이전 런과 달라야 한다
- **골드의 의미** — 지금은 숫자 구경이다
- **적의 위협감** — 일반 적이 너무 단순하다
- **방어의 가치** — 공격이 항상 우월한 구조

### 제일 먼저 할 것:
> **상인 노드 하나만 추가해도 게임이 완전히 달라진다.**

골드 소비처가 생기면: 골드 관리 → 카드 제거 → 덱 빌딩 → 런 전략 목표  
이 연쇄가 만들어진다. 단일 작업으로 임팩트가 가장 크다.
