# YCB 개선 체크리스트 관리

체크리스트 원본: `docs/GAME_DESIGN.md` 섹션 7 (## 7. 개선 체크리스트)

---

## 사용법

args에 따라 동작이 달라진다:

| 호출 방식 | 동작 |
|---|---|
| args 없음 | 전체 체크리스트 + 요약 출력 |
| `summary` | 티어별 완료/전체 카운트만 출력 |
| `add [티어] [항목 내용]` | 해당 티어에 새 항목 추가 |
| `done [부분 텍스트]` | 매칭되는 항목을 완료 처리 (✅ 날짜 추가) |
| `undone [부분 텍스트]` | 매칭되는 항목을 미완료로 되돌리기 |

티어 이름: `즉시` / `단기` / `중기` / `장기` / `코드` (코드품질)

---

## 실행 절차

### 1. 파일 읽기

`docs/GAME_DESIGN.md` 전체를 읽는다.  
섹션 7 (`## 7. 개선 체크리스트`) 부분을 파싱하여 티어별 항목을 추출한다.

티어 헤더 패턴:
- `### 🔴 즉시` — 즉시
- `### 🟡 단기` — 단기
- `### 🟢 중기` — 중기
- `### 🔵 장기` — 장기
- `### ⚙️ 코드 품질` — 코드품질

각 항목은 `- [ ]` (미완료) 또는 `- [x]` (완료) 형식이다.

---

### 2. args 없음 — 전체 출력

각 티어별로 항목을 나열한다. 완료 항목과 미완료 항목을 구분하여 출력한다.

출력 형식:
```
## 🔴 즉시 (완료 1 / 전체 3)
- [x] 상인 노드(MERCHANT) 추가 ✅ 2026-04-16
- [ ] 적 속성별 고유 행동 패턴 추가
- [ ] HEART 이벤트 재조정

## 🟡 단기 (완료 0 / 전체 7)
...

---
전체 완료: X / Y
```

---

### 3. `summary` — 카운트만 출력

```
🔴 즉시:   완료 1 / 3
🟡 단기:   완료 0 / 7
🟢 중기:   완료 0 / 3
🔵 장기:   완료 0 / 6
⚙️ 코드:   완료 0 / 5
─────────────────────
전체 완료: 1 / 24
```

---

### 4. `done [부분 텍스트]` — 완료 처리

1. 섹션 7에서 `부분 텍스트`가 포함된 미완료 항목(`- [ ]`)을 찾는다.
2. 여러 개 매칭되면 목록을 보여주고 사용자에게 선택 요청.
3. 하나 매칭되면: `- [ ] 항목` → `- [x] 항목 ✅ YYYY-MM-DD` 로 변경.
4. `docs/GAME_DESIGN.md`를 Edit 툴로 수정한다.
5. 변경된 항목과 갱신된 티어 카운트를 출력한다.

오늘 날짜는 시스템 날짜를 사용한다. 날짜를 모르면 `date` 명령을 실행해서 확인한다.

---

### 5. `undone [부분 텍스트]` — 미완료로 되돌리기

1. 완료 항목(`- [x]`)에서 부분 텍스트가 포함된 항목을 찾는다.
2. `- [x] 항목 ✅ YYYY-MM-DD` → `- [ ] 항목` 으로 변경 (✅ 날짜 제거).
3. `docs/GAME_DESIGN.md`를 Edit 툴로 수정한다.

---

### 6. `add [티어] [항목 내용]` — 항목 추가

1. 티어를 파싱한다: `즉시`→🔴, `단기`→🟡, `중기`→🟢, `장기`→🔵, `코드`→⚙️
2. 해당 티어의 마지막 항목 뒤에 `- [ ] [항목 내용]` 을 추가한다.
3. `docs/GAME_DESIGN.md`를 Edit 툴로 수정한다.
4. 추가된 항목과 티어 위치를 확인 출력한다.

티어 미지정 시 사용자에게 어느 티어에 추가할지 묻는다.

---

## 주의사항

- 파일 수정은 항상 `Edit` 툴로 한다 (`Write`로 전체 덮어쓰기 금지).
- 섹션 7 외 다른 섹션은 건드리지 않는다.
- 완료 날짜 형식은 `YYYY-MM-DD` (예: `2026-04-16`).
- 항목 텍스트 매칭은 대소문자 무관하게, 부분 문자열로 검색한다.
- 수정 후에는 반드시 수정된 항목을 명시적으로 출력하여 사용자가 확인할 수 있게 한다.
