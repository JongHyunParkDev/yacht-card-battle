# Phaser 개발 규칙

1. ts, eslint, prettier 에 맞게 개발한다.
2. 에셋 (이미지, 오디오 등) 파일이 추가되면 PreloadScene 에 추가한다.

- 이미지가 스프라이트 이미지 인 경우 파악해서 각 사이즈를 정교하게 분할할 수 있도록 width 와 height 사이즈를 주석을 적어준다.

3. 답변은 한국어한다.
4. 권한이 필요한 내용이면 반드시 물어보고 진행한다.
5. 마무리로 소스에 변경하기 전에 리팩터링을 하고 나서 적용한다.

- 너무 재활용이 안 된다. 프로젝트 context 를 이해 못 하는 것 같다.

6. 스타일은 왠만해서는 css 를 추가해서 사용하는 방식을 이용하고 조금 다른 경우에만 직접적으로 style 을 등록해 사용한다.
